Отчёт по 1 практической работе

Задание 1
В рамках первого задания была успешно разработана Use Case диаграмма для мобильного приложения "Трекер привычек". 
Диаграмма включает двух основных акторов - Гостя и Авторизованного пользователя, а также охватывает более 15 вариантов использования. 
Среди ключевых функций, отраженных на диаграмме: управление привычками (создание, редактирование, удаление), работа с категориями привычек, 
система напоминаний и целей, отслеживание статистики и прогресса, получение мотивационных цитат. 
Диаграмма была создана с соблюдением стандартов UML и включает все необходимые типы отношений между элементами.

Задание 2
Второе задание было посвящено проектированию архитектуры приложения согласно принципам Clean Architecture. 
Была разработана трехслойная структура, включающая Presentation Layer с 8 экранами приложения, Domain Layer с более чем 15 Use Cases и интерфейсами репозиториев, 
а также Data Layer с реализациями репозиториев и различными источниками данных. Были четко определены зависимости между слоями, 
при этом соблюдено основное правило Clean Architecture - зависимости направлены только внутрь, от внешних слоев к внутренним. 
Также была разработана схема потоков данных между слоями приложения.

Задание 3
Третье задание заключалось в практической реализации каркаса приложения на примере учебного проекта MovieProject. Были созданы все необходимые классы в соответствии с многослойной архитектурой: в Domain слое реализованы сущности Movie, Use Cases и интерфейсы репозиториев; в Data слое создана реализация MovieRepositoryImpl; в Presentation слое разработан MainActivity с функционалом сохранения и получения данных. Дополнительно была интегрирована работа с SharedPreferences для сохранения данных между сеансами приложения. Приложение было успешно запущено и протестировано на эмуляторе Android.

Задание 4
Четвертое задание предусматривало создание нового проекта "Трекер привычек" с применением полученных знаний. Был создан полноценный каркас приложения с соблюдением принципов Clean Architecture, реализованы основные Use Case классы для работы с привычками, настроены репозитории, возвращающие тестовые данные. Приложение успешно отображает список тестовых привычек и готово к дальнейшему расширению функциональности. Все классы корректно разделены по слоям, зависимости между слоями организованы согласно требованиям чистой архитектуры.
